# 친일반민족행위진상규명 보고서 — 기초 레이어 개요

> 정부(대통령소속 친일반민족행위진상규명위원회) 조사 결과 보고서.  
> 본 볼트의 **진실 기초 레이어(base layer)** 입니다. 모든 파생 노트는 이 원본에 출처가 연결됩니다.

## 1. 원본 책(2권 한 세트)

| 권 | 파일 | 용량 | 위치 |
|---|---|---|---|
| 전편 | `친일반민족행위진상규명_보고서_전편.pdf` | 약 619MB | `00-원자료/` |
| 후편 | `친일반민족행위진상규명_보고서_후편.pdf` | 약 312MB | `00-원자료/` |

## 2. OCR 처리 DB (원본 텍스트화 결과)

| DB | 대응 원본 | 비고 |
|---|---|---|
| `gemini_results.db` | 전편 | 전편 전체 페이지 |
| `gemini_results_vol4_p1.db` | 후편 전반 | 후편이 용량이 커 기술적으로 분할 처리 |
| `gemini_results_vol4_p2.db` | 후편 후반 | 후편의 나머지 |

> 후편이 DB 2개로 나뉘어 있는 것은 **처리 편의상 분할**이며, 원본 책은 2권이 맞습니다.

## 3. 파생 노트(인물/기관/사건/지역 등)

- DB에서 추출된 인물/기관/사건/지역 등 노트는 이 `00-원자료`를 기준으로 `source` frontmatter를 달아 역추적이 가능합니다.
- 보조 사료(불망·파묘·충북 학생 항일 독립투쟁사 등)에서 파생된 노트는 `source_id`로 구분됩니다.

## 4. 보조 원자료

- `DB/` : OCR 결과 DB 3개
- `불망, 파묘, 충북 학생 항일 독립투쟁사.zip` : 보조 사료 원본 압축본

## 5. 이 레이어의 원칙

1. **원본은 절대 수정하지 않음.** 교정은 DB 계층에서만 수행.
2. 모든 파생 노트는 `source` frontmatter로 출처를 남김.
3. 같은 인물·기관이라도 서로 다른 책에서 나오면 `source_id`로 구분.
4. 지역 조사 보고서(강원·충남·충북·전북·광주·춘천 등)는 별도 보조 레이어로, 본 보고서와 교차참조.
