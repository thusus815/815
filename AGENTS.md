# 친일반민족행위 진상규명 볼트 — 에이전트 가이드

> **이 문서는 LLM(Claude / Cursor / Codex / 기타 에이전트)이 이 볼트를 다룰 때 가장 먼저 읽는 스키마/규칙 문서다.**
> 사람의 작업 매뉴얼은 [`_가이드/마스터_인덱스`](_가이드/마스터_인덱스.md) 를 본다.

---

## 1. 이 볼트의 정체

- 정부(대통령소속 친일반민족행위진상규명위원회) **「친일반민족행위 진상규명 보고서」**(전편·후편)를 1차 사료로,
- **불망**, **파묘**, **충북 학생 항일 독립투쟁사** 등 보조 사료와
- **광주·강원·충남 등 지역별 친일·항일 보고서**를 통합한 한국 근대사 인물·사건·기관 위키.
- Andrej Karpathy의 LLM Wiki 패턴을 따른다. **벡터 DB·청킹·임베딩을 쓰지 않는다.** LLM이 인덱스부터 위키 링크를 따라 직접 탐색한다.

## 2. 3계층 구조 (Layer)

| 계층 | 폴더 | 역할 | LLM 권한 |
|---|---|---|---|
| **L1 원자료(raw)** | `00-원자료/`, `99-attachments/` | 1차 사료 PDF, OCR DB, 보조 사료 ZIP. **수정 금지**. | 읽기 전용. 절대 새 노트 생성/링크 금지. |
| **L2 위키(wiki)** | `01-인물/`, `02-사건/`, `03-단체/`, `04-지역/`, `05-문헌/`, `06-서훈/`, `07-항일학교/` | LLM·사용자가 만든 위키 노트. 자유 링크. | 읽기·생성·수정·링크 모두 가능. |
| **L3 메타(meta)** | `_가이드/`, `08-스냅샷/`, 루트의 `AGENTS.md` | 작업 규칙, 인덱스, 가이드, 시점 스냅샷. | 읽기·수정 가능. **그래프에서는 배제.** |

> 그래프 뷰는 L1·L3을 자동으로 가린다(`.obsidian/graph.json` 참조).

## 4. 노트 작성 규약

### 4-1. 모든 위키 노트의 공통 frontmatter

```yaml
---
type: 인물 | 사건 | 단체 | 지역 | 문헌 | 학교 | 서훈
side: 친일 | 항일 | 중립        # 인물/단체일 때만
source: 친일반민족행위진상규명_보고서   # 1차 출처
source_part: 전편 | 후편 | 전편·후편   # 어느 권에서 왔는지
출처_볼륨: vol1 ~ vol10           # OCR DB 권 번호
시작페이지: <int>
끝페이지: <int>
생성일: YYYY-MM-DD
tags: [...]
---
```

### 4-2. 인물 노트 추가 frontmatter

```yaml
이름: 갈홍기
한자: 葛弘基
생년월일: 1906.04.14
사망년월일: 1989.08.25
관련인물: [상호, 이광수, ...]   # 양방향 링크가 될 인물들
관련기관: [경성부, 동아일보, ...]
언급횟수: 3
```

### 4-3. 본문 표준 섹션 (인물 기준)

```markdown
# 이름 (한자, 생년 ~ 사망년)

## 가. 주요경력
## 나. 친일반민족행위 개요  (또는: 항일운동 개요)
## 다. 조사내용
## 라. 판단

## 관련 인물
- [[홍길동]]
## 관련 기관
- [[조선총독부]]
## 관련 사건
- [[3·1운동]]

## 새로 발견된 연결      # 단방향만 발견된 후보
## 상호 참조된 인물      # 양방향 확정된 인물
## 출처
- 친일반민족행위진상규명 보고서 vol7
- 페이지: 25~52 (28페이지)
```

## 5. 링크 규칙

1. **위키 노트끼리만 `[[...]]` 양방향 링크**를 만든다.
2. **L1(원자료)로 가는 링크는 만들지 않는다.** 출처 표기는 frontmatter `source`/`출처_볼륨`/`페이지`로 한다.
3. 본문 인용 안의 `[[이광수]]` 같은 링크는 **자동 ghost node를 만들 수 있으므로**, 해당 인물 노트가 없다면 링크 대신 일반 텍스트를 권장한다. 단, 추후 생성될 후보로 남기고 싶다면 `## 새로 발견된 연결` 섹션에 명시한다.
4. **태그**: `친일인물`, `항일인물`, `관료`, `언론`, `종교`, `교육`, `경제`, `군` 등 분류 태그와 `연도/1919`처럼 시대 태그를 쓴다.
5. **태그는 그래프에 노드로 띄우지 않는다** (`showTags: false`).

## 6. 에이전트 워크플로우 3종

### 6-1. **Ingest (원자료 → 위키)**
1. `00-원자료/DB/*.db` 또는 PDF의 특정 인물 구간을 읽는다.
2. 위 4-3 표준 섹션에 맞춰 `01-인물/<이름>.md` 생성.
3. 본문에서 등장한 인물·기관·사건을 `관련인물`·`관련기관`·`관련사건` frontmatter에 채우고, 동시에 노트가 존재하는 것만 본문에 `[[ ]]` 링크.
4. 양방향 확인: 이미 있는 상대방 노트가 이쪽을 언급하면 `## 상호 참조된 인물`에, 단방향이면 `## 새로 발견된 연결`에 기록.
5. 출처 정보 frontmatter 빠짐없이.

### 6-2. **Query (질문 → 답)**
1. **반드시 `_가이드/마스터_인덱스.md` 부터 읽고** 진입.
2. 인물/사건 단위 질문이면 해당 위키 노트로 진입 → `관련 *` 섹션을 따라 그래프 탐색.
3. 시기/주제 질문이면 `_가이드/그래프_레이어_가이드.md` 의 시나리오 활용.
4. 답변에는 항상 인용된 노트 경로와 원자료 페이지 번호를 함께 명시.

### 6-3. **Lint (위키 건강 점검)**
- 죽은 링크 (`[[X]]`인데 `X.md` 없음) 목록화 → `## 새로 발견된 연결` 로 이관.
- 같은 인물의 중복 노트 (이름·한자명 다른 표기).
- 출처 frontmatter 누락 노트.
- 그래프상 고립 노트 (`showOrphans: false`로 시각화는 가렸지만 점검은 해야 함).
- 결과는 `_가이드/lint_<날짜>.md` 로 저장.
- **자동화 도구**: `_가이드/scripts/lint_vault.py` — 위 모든 항목 점검 + 죽은 링크 Top 100 우선순위 정렬. 인자 없이 실행.

### 6-4. **Summarize (인물 노트에 요약 블록 추가)**
- 각 인물 노트의 frontmatter 직후에 `<!-- LLM-WIKI:SUMMARY:START --> ... <!-- LLM-WIKI:SUMMARY:END -->` 블록 삽입.
- 블록 내용: `## 한 줄 요약` + `## 핵심 행적` (3개) + `## 같은 흐름의 인물`.
- **원본 본문은 절대 변경하지 않는다**. 마커 블록만 추가.
- 도구: `_가이드/scripts/summarize_persons.py` (Gemini API 사용). 프롬프트는 `_가이드/scripts/prompts/summarize_person.md`.
- 사용법은 `_가이드/scripts/README.md` 참조.

## 7. 절대 금지

- ❌ `00-원자료/` 안의 어떤 파일도 수정/이동/삭제하지 않는다.
- ❌ `00-원자료/` 안에 새 노트를 만들지 않는다 (예외: `원자료_개요.md` 유지보수만).
- ❌ 사람 이름·한자·생몰년 등 frontmatter를 추측으로 채우지 않는다. 출처에 없으면 비워둔다.
- ❌ 정치적 평가나 현대적 해석을 본문에 임의로 추가하지 않는다. 보고서 원문의 사실 관계만 옮긴다.
- ❌ 첨부 PDF를 다른 폴더로 옮기지 않는다 (잠금/사료 무결성 보호).

## 8. 그래프 뷰 규칙 (자동화됨)

`.obsidian/graph.json` 에 다음이 적용되어 있다.

- `hideUnresolved: true` — 죽은 링크 ghost 노드 숨김
- `showOrphans: false` — 고립 노트 숨김
- `showTags: false` — 태그 노드 숨김
- `showAttachments: false` — PDF/이미지 숨김
- `search: -path:"00-원자료" -path:"_가이드" -path:"99-attachments" -path:"08-스냅샷"` — 메타·원자료 그래프 배제
- `colorGroups`: 친일(빨강) · 항일(파랑) · 학교(보라) · 단체(금) · 사건(진빨강) · 지역(회색) · 문헌(청록) · 서훈(주황)

색깔 또는 필터를 바꾸고 싶다면 이 파일을 수정한다.

## 9. 운영 스크립트

`_가이드/scripts/` 폴더에 다음 도구가 있다. 자세한 사용법은 같은 폴더의 `README.md`.

| 스크립트 | 기능 | API 필요 |
|---|---|---|
| `lint_vault.py` | 죽은 링크/고립/중복/요약 누락 점검 → `_가이드/lint_<날짜>.md` 생성 | ❌ |
| `summarize_persons.py` | 인물 노트에 LLM 요약 블록 자동 삽입 | ✅ Gemini |

```powershell
# 표준 작업 순서
cd "C:\Users\ho\Desktop\친일반민족행위진상규명_보고서\_가이드\scripts"
python lint_vault.py                                   # 1. 현재 상태 진단
python summarize_persons.py --dry-run                  # 2. 처리 대상 확인
python summarize_persons.py --execute --limit 5        # 3. 시험 실행
python summarize_persons.py --execute                  # 4. 전체 실행
python lint_vault.py                                   # 5. 변화 확인
```
