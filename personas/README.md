# 페르소나 시스템 안내

## 1. 설계 원칙

이 폴더의 페르소나는 **역사 자료를 기반으로 한 교육·연구용 대화 시스템**입니다.

- **실존 친일파 1인칭 페르소나는 만들지 않습니다.** (후손 명예훼손·역사 왜곡·면책 회피 등 위험 회피)
- **항일 인사도 1인칭으로 발화시키지 않습니다.** 단, 자료에 기록된 그분들의 실제 행적·발언은 인용 가능.
- 1인칭으로 말하는 페르소나는 **가상 인물(대표적 합성 인물)** 만 사용합니다.
- 객관 해설은 **큐레이터 페르소나**가 담당합니다. (3인칭, 학술 톤)

## 2. 페르소나 종류

| 파일 | 종류 | 시점 | 역할 |
|------|------|------|------|
| `박순길_청주농고_3학년.md` | 가상 학생 (1인칭) | 1930년 1월 ~ 1932년 졸업 | 충북 청주 항일 학생운동 체험 증언, 친일 인사와의 가상 대화 |
| `큐레이터_지역사건.md` | 큐레이터 (3인칭) | 전 시기 | 지역 사건·조직·인물 객관 해설, 사료 비교 |
| `_공통_방어선.md` | 공통 모듈 | — | 모든 페르소나에 prepend되는 5단계 방어선 |

## 3. 5단계 방어선 (요약)

1. **가상 인물 면식고지**: 가상 페르소나는 첫 대화·요청 시 가공 인물임을 밝힙니다.
2. **시대 정합성**: 페르소나의 활동 시기 이후 사건은 "직접 보지 못했다"고 답합니다.
3. **지역 정합성**: 활동 지역 외 정보는 "들은 풍문" 또는 "잘 모른다"고 답합니다.
4. **출처 표기**: 자료 기반 답변에는 답변 끝에 `[출처: 사료명]`을 붙입니다.
5. **실존 인물 보호**: 실존 인물의 미확인 발언·감정을 단정하지 않고, 기록된 행적만 인용합니다.

자세한 프롬프트는 `_공통_방어선.md` 참조.

## 4. 새 페르소나를 추가할 때

1. 가공 인물인지 큐레이터인지 결정 (실존 인물 1인칭 금지)
2. frontmatter에 다음 필드 작성:
   - `persona_id`, `type`, `era`, `region`, `scope`, `disclaimer`
3. 본문에 background, 말투, 어휘, 답변 예시 포함
4. Worker 코드에 `persona_id` 등록 (`worker/src/personas.ts`)

## 5. 사용처

- 사이트 `chat.html`에서 페르소나 선택 → Cloudflare Worker `/persona` 엔드포인트 호출
- Worker가 `personas/<id>.md` + `_공통_방어선.md`를 system prompt로 합성
- 그래프 RAG 컨텍스트 + 페르소나 prompt → Workers AI 응답
