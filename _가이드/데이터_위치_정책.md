# 데이터 위치 정책

> 이 프로젝트의 데이터를 **어디에 저장하고, 어디로 백업하고, 왜 그렇게 분배했는지** 한 장으로 정리한 문서.

## 한 줄 원칙

> **"모든 1차 입력 데이터는 C 드라이브가 마스터. D 드라이브는 백업. 그 외 분배는 명확한 이유가 있을 때만."**

---

## 1. 1차 입력 데이터 — 무조건 C 드라이브 마스터

| 데이터 | C 마스터 위치 | 비고 |
|---|---|---|
| 정부 보고서 PDF (전·후편) | `00-원자료/*.pdf` | 1GB. GitHub 업로드 불가. C가 유일 사본. |
| 보조 사료 ZIP | `00-원자료/불망_파묘_충북학생항일.zip` | |
| 정부 보고서 첨부 이미지 | `99-attachments/` | |
| **사용자 제출 이미지** | `00-원자료/사용자제출/images/` | R2에서 매일 동기화 |
| **사용자 제출 폼 본문** | `00-원자료/사용자제출/issues/` | GitHub Issues에서 매일 동기화 |
| **제출 비밀번호 키 목록** | `00-원자료/사용자제출/kv_snapshots/` | KV에서 매주 동기화 (해시 값은 보안상 미저장) |

> 외부(R2/GitHub/KV)에 있는 데이터도 **C가 마스터**라고 간주한다. 외부는 "운영용 사본".

## 2. D 드라이브 — C의 정기 백업

```
D:\과거사_백업\
├── daily\<YYYY-MM-DD>\          ← 매일 03:00, 7일치 보관
├── weekly\<YYYY>-W<주차>\        ← 매주 일요일, 4주 보관
├── snapshots\<YYYY-MM>\          ← 매월 1일, 무제한 보관
└── logs\                          ← 실행 로그
```

**백업 대상**:
- `00-원자료/` 전체 (정부 보고서 PDF + 사용자 제출 미러)
- `99-attachments/`
- 모든 위키 폴더 (`01-인물/` ~ `08-스냅샷/`, `_가이드/`, `_inbox/`)
- `chinil-db/` (SQLite DB 포함)

**제외**:
- `node_modules/`, Quartz 빌드 산출물 (재생성 가능)
- `quartz-815/`, `vault-wiki/` (Quartz 클론 — 재생성 가능)

## 3. 그 외 데이터 분배 — 명확한 이유

### ☁️ GitHub (`thusus815/815`) — 협업·배포 원천

| 무엇이 | 왜 GitHub에 |
|---|---|
| 위키 .md 1,654개 | 버전관리 + 다른 사람과 협업. GitHub Pages 빌드 원천. |
| `graph_data.json`, `graph.gexf` | Pages가 정적 호스팅. Worker가 fetch. |
| `chinil-db/*.db` (78MB×3 + 1.9MB) | 100MB 한도 안. 협업자가 clone으로 받음 = 추가 백업. |
| `.github/workflows/*.yml` | Actions 자동 배포에 필수. |
| `worker/src/*.ts` | Cloudflare Worker 소스. |

**GitHub에 안 올라가는 것**: 정부 보고서 PDF (용량 초과), R2 이미지 (R2가 운영처).

### ☁️ Cloudflare R2 (`archive-attachments`) — 사이트가 직접 참조하는 이미지 호스팅

- 사이트(`submit.html`, 노트 안의 이미지)가 R2 공개 URL을 직접 link.
- 이미지를 GitHub에 올리면 Pages 한도(레포 1GB / 트래픽 100GB월) 압박.
- R2는 트래픽 무료 + 10GB 무료.
- **C로 매일 동기화**되므로 R2가 죽어도 데이터 손실 없음.

### ☁️ Cloudflare Workers AI / KV — 동적 처리만

| 무엇이 | 왜 Cloudflare에 |
|---|---|
| Workers AI (llama-3.3-70b) | 챗봇 응답 생성 — 데이터 저장 X, 추론만. |
| KV `SUBMIT_AUTH` | 제출 비밀번호 해시 (30일 TTL). 작고 빠른 키-값. SQLite 띄울 필요 X. |

**Cloudflare D1 (SQL)은 사용 안 함** — 모든 구조화 데이터는 SQLite 파일 + 위키 .md로 충분.

## 4. 데이터 흐름 한 컷

```
[외부 입력]
   ├─ 정부 PDF       ──→ C:\00-원자료\*.pdf  (단방향, 변경 없음)
   ├─ 사용자 제출    ──→ Cloudflare R2 + GitHub Issue + KV
   │                       │
   │                       ├─ 매일 03:00 동기화
   │                       ▼
   │                    C:\00-원자료\사용자제출\
   │
[C 드라이브 (마스터)]
   │
   ├─ 매일 03:30 ──→ D:\과거사_백업\daily\
   ├─ 매주 일요일 ──→ D:\과거사_백업\weekly\
   └─ 매월 1일   ──→ D:\과거사_백업\snapshots\

[배포]
   C:\위키 .md  ──git push──→ GitHub  ──Actions──→ GitHub Pages
                                          │
                                          └──→ graph_data.json
                                                     ▲
                                          Cloudflare Worker가 fetch
```

## 5. 백업 스크립트

`_가이드/scripts/backup/` 아래:

| 스크립트 | 호출 빈도 | 역할 |
|---|---|---|
| `pull_issues.ps1` | 매일 | GitHub Issues → `00-원자료/사용자제출/issues/` |
| `pull_r2.ps1` | 매일 | R2 공개 URL → `00-원자료/사용자제출/images/` |
| `snapshot_kv.ps1` | 주 1회 | KV 키 목록 → `00-원자료/사용자제출/kv_snapshots/` |
| `mirror_to_d.ps1` | 매일 | C → D robocopy 미러 + 회전 |
| `run_backups.ps1` | 매일 (마스터) | 위 4개를 순서대로 호출. **스케줄러는 이것만 등록.** |

### Windows 작업 스케줄러 등록

관리자 PowerShell에서 한 번만:

```powershell
$action  = New-ScheduledTaskAction -Execute 'powershell.exe' `
  -Argument '-NoProfile -ExecutionPolicy Bypass -File "C:\Users\ho\projects\과거사 청산 프로젝트\친일반민족행위진상규명_보고서\_가이드\scripts\backup\run_backups.ps1"'
$trigger = New-ScheduledTaskTrigger -Daily -At 3:00am
$settings = New-ScheduledTaskSettingsSet -StartWhenAvailable -RunOnlyIfNetworkAvailable
Register-ScheduledTask -TaskName '과거사_일일백업' -Action $action -Trigger $trigger `
  -Settings $settings -RunLevel Highest -Description '매일 03:00 — Cloudflare 동기화 + D 미러'
```

확인:
```powershell
Get-ScheduledTask -TaskName '과거사_일일백업'
# 즉시 1회 테스트
Start-ScheduledTask -TaskName '과거사_일일백업'
```

## 6. 안전 점검 리스트 (월 1회)

- [ ] D:\과거사_백업\daily\ 에 최근 7일치 폴더가 다 있는가?
- [ ] D:\과거사_백업\weekly\ 에 4주치가 있는가?
- [ ] D:\과거사_백업\snapshots\ 에 이번 달 폴더가 생겼는가?
- [ ] C:\00-원자료\사용자제출\images\ 가 R2 버킷 객체 수와 일치하는가?
- [ ] C:\00-원자료\사용자제출\issues\ 에 이슈 번호 누락이 없는가?
- [ ] D 드라이브 여유 공간 100GB 이상 남았는가?
- [ ] C 드라이브 여유 공간 20GB 이상 남았는가?
