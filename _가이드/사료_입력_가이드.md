# 사료 입력 가이드

이 볼트에 새 자료를 추가하는 3가지 방법.

---

## 방법 1 — Obsidian 웹 클리퍼 (권장)

1. Chrome/Edge에 [Obsidian Web Clipper](https://obsidian.md/clipper) 설치
2. Cursor에서 `_가이드/웹클리퍼_템플릿.json` 내용을 복사
3. 웹 클리퍼 설정 → 템플릿 추가 → JSON 붙여넣기
4. 국가기록원, 공훈전자사료관 등 페이지에서 클리퍼 실행
5. 팝업에서 **사료 유형, 문서번호, 신뢰도** 입력 후 저장
6. 파일이 `_inbox/`에 저장됨

---

## 방법 2 — 수동 입력

1. `_inbox/00_입력_템플릿.md`를 복사해 새 파일 생성
2. YAML frontmatter의 `source_*` 필드 모두 입력
3. `_inbox/` 루트에 저장

### 필수 입력 필드

| 필드 | 설명 | 예시 |
|------|------|------|
| `source_type` | 사료 유형 | `공훈전자사료관` |
| `source_name` | 자료명 | `독립유공자 공훈록 - 서상경` |
| `source_institution` | 소장·발행기관 | `국가보훈부` |
| `source_date` | 발행·작성일 | `1930-03-15` |
| `source_doc_id` | 문서번호/청구번호/ISBN/DOI | `CJA0000213` |
| `source_url` | 원문 URL (없으면 "없음") | `https://...` |
| `source_reliability` | 신뢰도 선언 | `●●●` |

> **사료 미확인 시 자동으로 `_inbox/검증필요/`로 이동됩니다.**

---

## 방법 3 — LLM 자동 처리 (고급)

LLM이 자료를 정리해 `_inbox/`에 올릴 때도 동일한 템플릿 사용.
`source_*` 필드를 LLM이 채워야 하며, 원문 URL이 없으면 `●○○` 이하.

---

## 처리 파이프라인 실행

```powershell
# 수동 실행
cd C:\Users\ho\chinil-db
python process_inbox.py

# 자동 감시 (백그라운드)
python watch_inbox.py
```

처리 결과:
- 사료 신뢰도 자동 판정 (`●●●` / `●●○` / `●○○`)
- 기존 인물·사건·지역 노드 자동 매칭 및 링크 삽입
- `status: 처리완료` 변경 후 `_inbox/처리완료/`로 이동
- 출처 미확인 → `_inbox/검증필요/`로 이동

---

## 신뢰도 기준 요약

| 등급 | 기준 |
|------|------|
| `●●●` | URL·문서번호로 원문 직접 접근 가능 |
| `●●○` | 기관·날짜·제목 명확 (원문 미확인) |
| `●○○` | 출처 있으나 2차 인용 |
| `○○○` | 출처 불명 → **입력 불가, 검증필요 이동** |
