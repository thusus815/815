# 검토자 운영 가이드 (review.html)

> 자료 제출 검토를 외부 검토자(김남균 선생 등)에게 위임하기 위한 운영 절차.

## 1. 권한 분리

| 역할 | 화면 | 무엇을 할 수 있나 | 무엇을 할 수 없나 |
|---|---|---|---|
| 제출자 (누구나) | `submit.html` | 자료 업로드 + 본인 이슈에 댓글 | 다른 자료 보기 / 승인 |
| **검토자 (김남균 선생 등)** | **`review.html`** | 자동 분류 미리보기 보기 / 의견 코멘트 / 4종 플래그 | 승인·반려·커밋·삭제 |
| 관리자 (운영자 본인) | `admin.html` | 모든 권한 (승인·반려·OCR·이슈 close) | - |

검토자는 GitHub 권한 일절 없음 — 워커가 검토자 명의로 GitHub Issue에 댓글·라벨만 부착.

## 2. 검토자 비밀번호 발급

비밀번호는 **검토자별로 각자 다른 값**을 발급해 누가 어떤 의견을 남겼는지 추적합니다.

### 2-1. 비밀번호 결정

각 검토자에게 강한 임의 문자열을 발급. 예시 (실제로는 새로 생성):
```
김남균 → kim_8Wq7zR2nT4mPx5L
홍길동 → hong_9Yk2cV6bN3sQa1H
```

### 2-2. 워커 시크릿 등록

콤마로 구분, `이름:비밀번호` 형식으로 한 번에 등록:

```powershell
cd "C:\Users\ho\projects\과거사 청산 프로젝트\친일반민족행위진상규명_보고서\worker"

# 윈도우는 echo가 BOM/줄바꿈을 넣을 수 있어 wrangler에 직접 입력하는 방식 사용
npx wrangler secret put REVIEWER_SECRET
# 프롬프트가 뜨면 입력: 김남균:kim_8Wq7zR2nT4mPx5L,홍길동:hong_9Yk2cV6bN3sQa1H
```

### 2-3. 검토자 추가/제거

같은 명령으로 다시 입력하면 덮어쓰기. 한 명만 발급할 거면:
```
김남균:kim_8Wq7zR2nT4mPx5L
```

해제는:
```powershell
npx wrangler secret delete REVIEWER_SECRET
```

## 4. 운영자 측 처리 흐름

검토자 의견이 들어오면 운영자는:

1. **GitHub Issues** 에서 라벨로 필터:
   - `검토완료` → admin.html에서 빠르게 일괄 승인 가능
   - `수정필요` → 의견을 보고 `md_override`로 수정 후 admin.html에서 승인
   - `분류재검토` → admin.html에서 `folder_override`로 폴더 변경 후 승인
   - `반려추천` → admin.html에서 반려 처리

2. **`admin.html` 의 카드** 에는 검토 라벨이 자동 표시되도록 향후 개선 예정 (현재는 GitHub 이슈에서 라벨 직접 확인).

## 5. KV 상태 — `REVIEW_STATE` 네임스페이스

| 키 패턴 | 값 | TTL |
|---|---|---|
| `session:{token}` | `{name, expires}` | 7일 |
| `review:{issue_number}` | `{status, comments[], last_reviewer, updated_at}` | 무한 |

수동 조회 (디버깅):
```powershell
cd worker
npx wrangler kv key list --binding REVIEW_STATE --remote
npx wrangler kv key get --binding REVIEW_STATE --remote "review:26"
```

## 6. API 요약 (다른 검토 도구 만들 때 참조)

| 메서드 | 경로 | 인증 | 페이로드 |
|---|---|---|---|
| POST | `/review/login` | (비밀번호) | `{password}` → `{token, reviewer_name}` |
| GET | `/review/issues` | `Bearer <token>` | - → `{issues: [...]}` (자동 분류 미리보기 포함) |
| POST | `/review/comment` | `Bearer <token>` | `{issue_number, text}` |
| POST | `/review/flag` | `Bearer <token>` | `{issue_number, status, note?}` (status: ok/edit_needed/reject_suggested/reroute_suggested) |
